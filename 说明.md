# CellPose 项目说明

## 1. 项目是做什么的

本项目用于细胞图像分割，目标是对输入的显微图像进行实例分割或语义分割，并输出掩膜、可视化叠加图以及研究实验报告。

项目包含两条主线：

1. `main.py`
   基于 Cellpose 的现成分割模型做本地推理，适合直接对细胞图像进行分割。
2. `run_cellpose.py --study`
   基于 PyTorch 的研究流水线，围绕小型 UNet 及注意力模块进行训练、评估、可视化、测速和报告生成。

简而言之，这个系统既能作为一个可直接运行的细胞分割工具，也能作为一个“注意力机制细胞分割研究框架”使用。

## 2. 系统使用了什么模型

### 2.1 Cellpose 推理模型

`main.py` 默认调用 `CellposeModel`，默认模型参数为 `cpsam`。

代码中会按以下顺序寻找模型：

1. 用户通过 `--model` 显式指定的模型路径或模型名
2. 项目本地的 `models/cpsam`
3. 用户目录下的 `.cellpose/models/cpsam`
4. 如果都不存在，则尝试按 Cellpose 规则下载

因此，项目面向直接推理时的核心模型是：

- `Cellpose`
- 默认预训练模型：`cpsam`

### 2.2 研究模型

研究模块位于 `research/` 目录，核心模型是一个小型 `UNet`，并支持四种实验配置：

- `baseline`
- `se`
- `cbam`
- `se_cbam`

其中：

- `baseline`：不加注意力模块的基础 UNet
- `se`：在卷积块后加入 SE 注意力
- `cbam`：在卷积块后加入 CBAM 注意力
- `se_cbam`：先 SE 再 CBAM 的串联注意力结构

对应实现位置：

- UNet 模型：[research/models/unet.py](D:\program_for_customer\BiShe\CellPose\research\models\unet.py)
- 注意力模块：[research/models/attention.py](D:\program_for_customer\BiShe\CellPose\research\models\attention.py)
- 模型构建入口：[research/engine/common.py](D:\program_for_customer\BiShe\CellPose\research\engine\common.py)

## 4. 目录功能概览

- `main.py`：Cellpose 本地推理入口
- `run_cellpose.py`：统一入口，可走普通推理或研究流水线
- `configs/`：实验配置文件
- `data/`：toy 数据集、正式数据集占位目录
- `images/`：默认输入图像目录
- `outputs/`：Cellpose 推理输出目录
- `research/`：研究代码，包括数据、模型、训练与评估逻辑
- `reports/`：实验汇总、可视化结果和报告草稿
- `scripts/`：各阶段 PowerShell 脚本
- `models/`：本地模型文件目录，默认可放 `cpsam`

## 5. 如何运行系统

### 5.1 环境准备

本项目至少需要一个 Python 环境。

基础依赖可先安装：

```powershell
pip install -r requirements.txt
```

如果要运行研究流水线，还需要安装 `PyTorch`。

例如：

```powershell
pip install torch torchvision
```

如果要运行 Cellpose 推理，还需要安装 `cellpose`。

例如：

```powershell
pip install cellpose
```

注意：

- 当前 [requirements.txt](D:\program_for_customer\BiShe\CellPose\requirements.txt) 只列出了基础依赖，没有把 `torch` 和 `cellpose` 写进去。
- `main.py` 中默认会尝试切换到 `D:\Tools\Anaconda\envs\cellpose\python.exe` 运行。如果你的机器没有这个环境，建议自行安装 `cellpose`，或者设置环境变量 `CELLPOSE_PYTHON_EXE` 指向正确的 Python。

### 5.2 运行 Cellpose 细胞分割

最直接的方式：

```powershell
python main.py --input ./images --output ./outputs
```

如果要处理单张图片：

```powershell
python main.py --input ./images/cellpose.png --output ./outputs
```

如果要启用 GPU：

```powershell
python main.py --input ./images --output ./outputs --gpu
```

常见参数：

- `--model`：模型名或模型路径，默认 `cpsam`
- `--diameter`：细胞直径
- `--batch-size`：推理批大小
- `--flow-threshold`：mask 质量控制阈值
- `--cellprob-threshold`：细胞概率阈值
- `--recursive`：递归搜索输入目录中的图像

运行后会在 `outputs/` 中生成：

- `*_cp_masks.tif`
- `*_cp_masks.npy`
- `*_cp_flows.npy`
- `*_cp_overlay.png`

### 5.3 使用统一入口运行普通推理

`run_cellpose.py` 默认行为是转发到 `main.py`，并固定输出到 `outputs/`：

```powershell
python run_cellpose.py --input ./images
```

### 5.4 运行研究流水线

如果要执行研究模式，使用：

```powershell
python run_cellpose.py --study
```

这条命令会按配置执行一套完整流程，包括：

- 生成或检查 toy 数据集
- 训练 baseline / SE / CBAM / SE+CBAM 模型
- 生成可视化结果
- 进行推理测速
- 汇总消融实验结果
- 生成 Markdown 报告

更完整的示例：

```powershell
python run_cellpose.py --study --generate-toy-data --force-train
```

研究模块也可以拆开单独运行：

```powershell
python -m research.datasets.generate_toy_dataset --output data/toy_cells
python -m research.engine.train --config configs/baseline_toy.yaml
python -m research.engine.infer --config configs/baseline_toy.yaml --checkpoint <checkpoint_path> --input data/toy_cells/test/images
```

## 6. 运行结果会输出到哪里

- Cellpose 推理结果默认输出到 `outputs/`
- 研究训练权重默认输出到 `runs/`
- 实验报告和可视化默认输出到 `reports/`

## 7. 当前项目的定位

这个项目不是单纯的 Cellpose 调用脚本，而是一个面向“细胞分割 + 注意力机制研究”的课程/研究型工程。

它的核心目的包括：

- 对细胞图像进行自动分割
- 研究 SE、CBAM 等注意力机制对分割效果的影响
- 建立从训练、评估、可视化到报告生成的完整闭环
- 为后续接入真实数据集和开展正式实验提供基础框架

## 8. 推荐的最小运行路径

如果你只想快速验证系统可用，建议按这个顺序：

1. 安装基础依赖
2. 安装 `cellpose`
3. 将待分割图片放入 `images/`
4. 运行：

```powershell
python main.py --input ./images --output ./outputs
```

如果你要验证研究模块，建议运行：

```powershell
python run_cellpose.py --study --generate-toy-data
```
